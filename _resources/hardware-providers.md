A snapshot of who is building what, organized by qubit modality. Specs change
quickly — always cross-check the vendor's current docs.

## Superconducting

- [**IBM Quantum**](https://quantum.ibm.com/) — Heron / Condor families;
  free-tier cloud access via the IBM Quantum Platform.
- [**Google Quantum AI**](https://quantumai.google/hardware) — Sycamore and
  successor chips (limited external access).
- [**Rigetti**](https://www.rigetti.com/) — Ankaa-series; available via QCS,
  AWS Braket, and Azure Quantum.
- [**IQM**](https://www.meetiqm.com/) — On-premise and cloud-accessible
  superconducting systems.

## Trapped ion

- [**IonQ**](https://ionq.com/) — Forte / Tempo systems via AWS Braket, Azure
  Quantum, GCP.
- [**Quantinuum**](https://www.quantinuum.com/) — H-series with
  mid-circuit measurement and qubit reuse; cloud access via own API and Azure.

## Neutral atom

- [**QuEra**](https://www.quera.com/) — Aquila (analog Rydberg) on AWS Braket.
- [**Pasqal**](https://www.pasqal.com/) — Analog and digital neutral-atom
  processors.
- [**Atom Computing**](https://atom-computing.com/) — Large-scale neutral-atom
  arrays.

## Photonic

- [**Xanadu**](https://www.xanadu.ai/) — Borealis and successors via Xanadu
  Cloud; PennyLane integration.
- [**PsiQuantum**](https://www.psiquantum.com/) — Pursuing fault-tolerant
  photonic architecture.

## Cloud aggregators

- [**AWS Braket**](https://aws.amazon.com/braket/), [**Azure
  Quantum**](https://azure.microsoft.com/products/quantum), [**Google
  Cloud**](https://cloud.google.com/quantum-ai) — multi-vendor access layers.
